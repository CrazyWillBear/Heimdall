Create the Heimdall GitHub App

One click creates a GitHub App with exactly the permissions Heimdall needs (Pull requests: read & write, Contents & Metadata: read) subscribed to pull_request events, with its webhook pointed at your deployment.

The public host where the web service is reachable. The webhook URL becomes https://<domain>/webhook.

After you click Create

  1. GitHub creates a temporary app and redirects your browser to https://<domain>/?code=<CODE>. A 404 page is fine — copy the code value from the address bar.
  2. Exchange it for the real credentials:
    gh api -X POST /app-manifests/<CODE>/conversions
  3. From the JSON response: put id in GITHUB_APP_ID and webhook_secret in WEBHOOK_SECRET (both in .env), and write pem to secrets/github_app_private_key.pem.
  4. Install the App on the repositories you want reviewed.
